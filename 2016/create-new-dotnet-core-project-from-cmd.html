由命令列建立 .NET Core 新專案

Aug 2, 2016 | 軟體建構 | Hits

在 Windows 下用 Visual Studio 內建的範本即可新增專案，在命令列下，則有二種新建專案的方式。

SDK 的 dotnet new 命令

dotnet new 帶有2個參數， -t 跟 -l

-t , –type

指定專案類型

dotnet new : 預設，Console 專案，也可用 dotnet new -t console
dotnet new -t web : ASP.NET Core Web 專案
dotnet new -t lib : Class Library
dotnet new -t xunittest : xUnit Unit test 專案

-l , –lang [C#][F#]

指定專案的程式語言，預設是 C#，有 C# F# 可選用

dotnet new --lang f#

Yeoman

Yeoman 是 Node.js 下自動產生網站骨架或程式碼的工具，配合開發好的 generator，
即可自動建立所需類型的專案或程式碼。

安裝 Yeoman

npm install -g yeoman

安裝 ASP.NET core generator

npm install -g generator-aspnet

要建立 dotnet core 專案，就輸入

yo aspnet

上下選擇專案類型後，按下 Enter，再選擇或填入必要資料後，即完成建立。
由於 yeoman 會自動執行 bower install 抓回必要套件，請耐心等待。

Yeoman 除了提供專案範本外，也提供 Sub Generators 功能可建立各類型的類別，下yo aspnet:Class MyClass即可新增名稱為 MyClass 的 Class。

generator aspnet 附帶有豐富的 sub generators，可提供類似於 Visual Studio 的類別範本，如 :

aspnet:appsettings
aspnet:bowerjson
aspnet:class
…

可參考這裡選用 : Sub Generators

編譯及執行

dotnet restore

dotnet restore透過 NuGet 去抓回必要的套件，並將資訊寫入 project.lock.json 供編譯和執行使用。

dotnet run

dotnet run先呼叫dotnet build確定完成編譯，再呼叫dotnet <assembly.dll>去執行。
